## 1. 项目背景与目标

### 1.1 背景

煤炭行业研究需要持续追踪大量中观指标，数据来源分散（煤炭网、中电联、海关总署、华通人、Bloomberg、Clarkson），频率不一（日度/周度/月度），现有方式依赖手工汇总 Excel，效率低，难以形成系统性研判。

### 1.2 目标

构建一个**单文件静态 Web 应用**，集成行业关键指标与外生关键指标的录入、可视化、比价分析与投资信号生成，部署于 GitHub Pages，零服务器成本。

## 2. 用户画像

**主要用户**：煤炭行业研究员 / 大宗商品策略分析师  
**使用场景**：  
- 每周一更新周度数据（价格、库存）后查看整体研判
- 每月20日前后更新月度数据（产量、进出口、外生指标）
- 临时查阅期货行情与比价关系
- 生成投资报告前汇总信号

**核心诉求**：快、准、一屏看全，不依赖后端，数据可本地留存

## 3. 功能需求

### 3.1 标签页架构（10个）

| # | 标签 | 核心功能 | 数据来源 |
|---|------|----------|----------|
| 1 | 总览 | 4个KPI卡片 + 供需简表 + 价格走势图 | 混合 |
| 2 | 供给端 | 产量/进口量/开工率完整表 | 模拟+手动录入 |
| 3 | 需求端 | 日耗/高炉开工率/发电量完整表 | 模拟+手动录入 |
| 4 | 库存端 | 港口/电厂/钢厂库存 + 走势图 | 模拟+手动录入 |
| 5 | 外生指标 | 表3外生指标全覆盖（9项）| 模拟+手动录入 |
| 6 | 比价关系 | 焦炭/焦煤比价、焦化利润、进口价差 | 期货衍生 |
| 7 | 期货行情 | ZC/J/JM实时行情 + 期限结构 + 套利 | 东方财富API |
| 8 | 上市公司 | 8只A股煤企实时行情 + 估值表 | 东方财富API |
| 9 | 数据录入 | 手动录入 + CSV批量导入 + 导出 | 本地IndexedDB |
| 10 | 投资信号 | 动态打分 + 综合研判文字 | 期货+录入数据 |

### 3.2 数据录入（F1）

**手动录入字段**（9项）：

| 字段ID | 中文名 | 单位 | 示例值 |
|--------|--------|------|--------|
| date | 录入日期 | — | 2025-06-01 |
| qhd | 动力煤（秦皇岛） | 元/吨 | 785 |
| coking | 炼焦煤（山西古交2#） | 元/吨 | 1300 |
| coke | 焦炭（山西太原一级） | 元/吨 | 1820 |
| inv | 秦皇岛港口库存 | 万吨 | 482 |
| cons | 六大电厂日耗 | 万吨 | 78 |
| prod | 全国原煤产量 | 万吨/月 | 38000 |
| imp | 进口煤总量 | 万吨/月 | 4200 |
| rebar | 螺纹钢 | 元/吨 | 3650 |

**CSV批量导入规格**：
- 支持中英文双语列名（`qhd_price` 或 `动力煤` 均可识别）
- 支持 UTF-8 / GBK 编码
- 导入前展示前8行预览，用户确认后写入
- 文件大小上限：5MB
- 支持拖拽 + 点击两种方式触发

**CSV模板字段**（英文列名）：
```
date, qhd_price, coking_price, coke_price, port_inventory,
power_consumption, raw_production, coal_import, rebar_price
```

### 3.4 投资信号（F3）

**打分维度**（4项，共100%）：

| 维度 | 权重 | 计算逻辑 |
|------|------|----------|
| 供给紧张度 | 25% | 产量同比 + 进口量同比 |
| 需求强度 | 30% | 日耗变化 + 高炉开工率 |
| 库存压力 | 25% | 港口库存绝对值 vs 历史均值 |
| 比价套利 | 20% | 焦化利润（J - JM×1.35 - 450） |

综合评分 = 各维度得分（满10分）× 权重求和，最终输出0-10分 + 方向研判文字。

### 3.5 存储（F4）

- **存储引擎**：IndexedDB（数据库名 `coal_tracker`，版本2）
- **Object Store**：`entries`（keyPath: `id`，autoIncrement）
- **数据结构**：见录入字段表，附加 `id`、`createdAt`、`source` 字段
- **操作**：新增、删除单条、清空全部、全量导出CSV

## 5. 已知限制与约束

1. **AKShare 不可在浏览器直调**：AKShare 是 Python 库，需通过 GitHub Actions 定期运行 → 输出 JSON 静态文件 → 前端读取（计划在 v4 阶段实现）。
2. **东方财富API不稳定**：非官方接口，可能被限流；必须有模拟数据兜底。
3. **期货非交易时段**：行情接口在非交易时段返回上一交易日收盘数据，属正常现象，无需报错。
4. **IndexedDB 清空风险**：浏览器私有/无痕模式下 IndexedDB 在窗口关闭后清除，需告知用户定期导出。
